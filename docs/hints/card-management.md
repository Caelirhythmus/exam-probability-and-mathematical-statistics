# 卡片管理

## 目录结构

```
docs/exam/md/1-collections/
├── cards/                # 卡文件（一版块一文件）
│   ├── 01-<版块>.md
│   ├── 02-<版块>.md
│   └── ...
├── deprecated/           # 旧版归档
├── scripts/              # 本地脚本入口（符号链接）
├── anki-cards.tsv        # 构建产物（被 .gitignore 排除）
└── not-release-collections.md  # 未发布的卡草稿
```

## 新增一个版块

1. 在 `cards/` 下创建 `<序号>-<版块名>.md`
2. 编号按已有文件递增，不插入已占用的序号
3. 格式：

```
主题：<版块>
求解：<入口问题>
路径：<答案>

主题：<版块>，需要<操作>
求解：<具体问题>
路径：<公式或方法>
```

## A/B 双卡约定

每对 A/B 卡写在一个空行隔开的块里，顺序先 A 后 B。

| 卡型 | 主题模板 | 求解角色 | 路径 |
|------|----------|----------|------|
| A | `版块` 或 `版块，上一步提示` | 决定下一步做什么 | 操作名称 |
| B | `版块，需要<操作>` | 问具体怎么做 | 公式或方法 |

入口 A 卡主题仅写版块名。

## 构建

```bash
# 从项目根目录运行
python scripts/anki/build-anki-cards.py
```

输出 `anki-cards.tsv` 生成在 `1-collections/` 下，直接导入 Anki。

## 规则

- 不直接编辑 `anki-cards.tsv`，它由脚本生成
- 卡片不够原子时可拆成两张，序号保持不变
- 不同版块的文件互不依赖，可独立编辑
