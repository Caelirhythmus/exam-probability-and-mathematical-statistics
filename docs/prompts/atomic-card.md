# 原子卡制作规范

## 格式

每条卡片为三行，依次是 `主题`、`求解`、`路径`，卡间空一行：

```
主题：<上一步已完成什么的提示>
求解：<当前步要解决的问题>
路径：<答案（玩家需回忆的内容）>
```

## A/B 双卡机制

每个原子步骤拆成 **A 卡** 和 **B 卡** 两张：

| 卡型 | 角色 | 主题 | 求解 | 路径 |
|------|------|------|------|------|
| **A 卡** | 决定下一步做什么 | 上一步完成后，提示环境 | 问下一步求什么值/做什么操作 | 告知操作名称 |
| **B 卡** | 知道做什么后问怎么做 | 直接写「需要<操作名>」 | 问具体求法/操作内容 | 公式或具体方法 |

### 示例

```
A: 主题：已有 α 和 df
   求解：下一步做什么
   路径：查临界值

B: 主题：需要查临界值
   求解：具体查哪个值
   路径：t_{α/2}(df)
```

### 规则

- A 卡与 B 卡成对出现，A 在前 B 在后
- 入口卡（第一个 A 卡）也配 B 卡
- 最后一步的 B 卡之后没有下一张 A 卡
- 整体构成完整流程：入口 A → B → 下一 A → B → ... → 出口

## 主题的上下文规则

- **A 卡主题**：只提示上一步已完成什么，不透露当前步怎么做
- **B 卡主题**：直接提示「需要<操作名>」，不需要上一步上下文
- **求解**：当前步面对的问题
- **路径**：答案（回忆项）

第一张 A 卡（入口卡）没有上一步，主题直接写入口场景。

## 原子性

- **够小、够细、够原子**：一张卡片只讲一个概念或一个步骤
- 不在一张卡里塞多个知识点
- 如果一步可以拆成两步，就拆成两张卡

## 制卡过程
先生成相应的例题和解答，然后再检查自己生成的内容，提取出流程线，分解出原子步骤，对每个原子步骤制作 A、B 两张卡。

## 示例

```
主题：某分布
求解：范围概率用什么工具
路径：切比雪夫不等式

主题：需要范围概率的下界
求解：用什么定理
路径：切比雪夫不等式

主题：工具选切比雪夫
求解：下一步先算什么
路径：期望 E(X) 和方差 D(X)

主题：需要算期望和方差
求解：具体怎么算
路径：连续型用 ∫ xf dx，离散型用 ∑ xp

主题：已有期望和方差
求解：下一步算什么
路径：区间概率的下界

主题：需要区间概率下界
求解：公式是什么
路径：P(|X-μ|<k) ≥ 1 - D(X)/k²
```

## 递进关系

- 多张卡按解题/推理顺序排列
- A → B → 下一个 A → B，前一张的输出是后一张的输入
- 整体构成一条完整的问题解决路径
