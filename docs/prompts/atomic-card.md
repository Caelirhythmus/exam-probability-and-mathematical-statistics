# 原子卡制作规范

## 格式

每条卡片为三行，依次是 `主题`、`求解`、`路径`，卡间空一行：

```
主题：<上一步已完成什么的提示>
求解：<当前步要解决的问题>
路径：<答案（玩家需回忆的内容）>
```

## 主题的上下文规则

- **主题**：只提示上一步已完成什么，不透露当前步怎么做
- **求解**：当前步面对的问题
- **路径**：答案（回忆项）

第一张卡（入口卡）没有上一步，主题直接写入口场景。

## 原子性

- **够小、够细、够原子**：一张卡片只讲一个概念或一个步骤
- 不在一张卡里塞多个知识点
- 如果一步可以拆成两步，就拆成两张卡

## 制卡过程
先生成相应的例题和解答，然后再检查自己生成的内容，提取出流程线，分解出原子步骤，并对每个步骤都制作一个卡片，使得卡片顺序就是步骤顺序。

## 示例

```
主题：某分布
求解：范围概率用什么工具
路径：切比雪夫不等式
```

```
主题：已确定用切比雪夫
求解：先算什么量
路径：期望和方差
```

```
主题：已有临界值
求解：还差什么
路径：标准误 s/√n
```

## 递进关系

- 多张卡按解题/推理顺序排列
- 前一张的输出是后一张的输入
- 整体构成一条完整的问题解决路径
